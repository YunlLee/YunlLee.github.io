1. 二叉树

树的基本概念

  1. 节点、根节点、父节点、子节点、兄弟节点。

  2. 一棵树可以没有任何节点，称为空树。

  3. 一棵树可以只有一个节点，这个时候这棵树只有根节点。

  4. 子树、左子树、右子树

  5. 节点的度：子树的个数

  6. 叶子节点：度为 0 的节点

  7. 非叶子节点：度不为 0 的节点

  8. 层数：根节点在第一层，根节点的子节点在第二层，以此类推。

  9. 节点的高度：当前节点到最远叶子节点的路径上的节点总数。

  10. 树的高度：所有节点高度中的最大值。

二叉树的特点

  1. 每个节点的度最大为2（即最多拥有两颗子树）.

  2. 左子树和右子树是有相对顺序的

  3. 非空二叉树的第 iii 层，最多有 2i−12^{i-1}2i−1

  4. 高度 hhh 的二叉树上对多有 2h−12^{h}-12h−1 个节点

满二叉树

最后一层的节点的度都为 0 ，其它节点度都为 2，在同样高度的二叉树中，满二叉树的叶子节点最多，总数节点最多。

假设满二叉树的高度为 hhh ，那么第 iii 层的节点数量： 2i−12^{i-1}2i−1 ，叶子节点的数量：2h−12^{h-1}2h−1 ，总节点的数量：2h−12^{h}-12h−1

完全二叉树

叶子节点只会出现在最后两层，且最后一层的叶子节点从左向右填充。

假设完全二叉树的高度为 h，那么至少有 2h−12^{h-1}2h−1 个节点，最多有 2h−12^{h} - 12h−1

二叉树的遍历

如果 是一棵有 个结点子树的根，那么遍历它的时间成本为 O(n)

证明：

  1. 空树的处理

作用于一棵空树时，函数会立即返回。这需要花费一个很小的常数时间（通常用于检查根节点是否为 NIL）。因此，对于一棵空树，我们可以设定时间复杂度 𝑇(0)=c𝑇(0)=cT(0)=c，其中 c 是一个常数。

  1. 递归公式

对于一棵非空的树，假设根节点的左子树有 kkk 个节点，右子树有 n−k−1n - k - 1n−k−1 个节点，则在根节点上的操作所需时间可以表示为：

T(n)=T(k)+T(n−k−1)+dT(n) = T(k) + T(n - k - 1) + d T(n)=T(k)+T(n−k−1)+d

其中, d 是处理当前节点所需的常数时间。这个公式反映了在根节点上执行操作的总时间。

  1. 替换法

假设时间复杂度的形式为：T(n)≤(c+d)n+cT(n) \le (c + d)n + cT(n)≤(c+d)n+c

对于 n=0n = 0n=0，我们已经有：T(0)=cT(0) = cT(0)=c

对于 n>0n > 0n>0, 我们有：T(n)≤T(k)+T(n−k−1)+dT(n) \le T(k) + T(n - k - 1) + dT(n)≤T(k)+T(n−k−1)+d

带入假设：

T(n)≤(c+d)k+c+(c+d)(n−k−1)+c+d=(c+d)n+cT(n) \le (c + d)k + c + (c + d)(n - k - 1) + c + d = (c + d)n + c T(n)≤(c+d)k+c+(c+d)(n−k−1)+c+d=(c+d)n+c

由此，可得出 T(n)=O(n)T(n) = O(n)T(n)=O(n)

先序遍历：根->左->右

中序遍历：左->根->右

后序遍历：左->右->根

查询

从树根开始递归期间遇到的结点就形成了一条向下的简单路径，所以 treeSearch 的运行时间为 O(h), 其中 h 为这棵树的高度。

我们可以采用 while 循环来展开递归，用一种迭代方式重写这个过程。对千大多数计算机，迭代版本的效率要高得多。
