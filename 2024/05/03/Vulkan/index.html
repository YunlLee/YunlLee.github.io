0. Vulkan 接口理论

0.1 概览vulkan接口设计

vulkan里的一个概念 Host & Device
Host: 一切以 CPU 和 内存 为基础的操作，都是 Host，比如模型读取，鼠标键盘事件响应，游戏碰撞和逻辑。
Device：一切由 GPC 和 显存 为基础进行的图形渲染、后处理等操作，都是 Device 端。

0.2 元数据与设备

Instance：用来表示渲染程序的元数据，比如vulkan版本，引擎版本，是否启用Debug，是否启用哪
些扩展等
Surface： 用来链接操作系统相关的窗口与Vulkan渲染出来的图像的组件
PhysicalDevice： 用来代表一张显卡，我们可以自己挑选使用哪一张显卡运行渲染
LogicalDevice： 显卡属于Device端，我们申请一个使用资源之后，用逻辑设备在Host端“远程”操作设备

0.3 交换链

假设我们有一块内存（帧数据），这块内存上存储着颜色数据，我们需要将内存中的数据显示到屏幕上。屏幕上都是像素，渲染像素是从左到右一行一行渲染，渲染每一行时，都会在内存中取数据。

水平消隐：扫描完毕一行，回到下一行开头的时间
垂直消隐：一页扫描显示完毕，回到左上角的时间

如果在垂直消隐来临之前，就去更新了帧数据的RGB值们，那么就会产生“撕裂”，乃至花屏。(显卡计算的速度时远远比屏幕刷新的速度要快)，所以必须等待垂直消隐时间，才能更新帧数据。
刷屏时间+渲染更新时间=一帧显示时间

双 buffer

如果使用两个Buffer，就可以做到：

  • 1 Buffer-1 用来被屏幕空间读取进行显示
  • 2 Buffer-2 用来被GPU 用来渲染
  • 3 当屏幕进入垂直消隐期间，就可以交换两个帧缓冲，

从而实现无缝对接与并行工作
这个过程就是交换链（Swap Chain）进行管理的

vulkan 是一个最小驱动上层上层 API，所以像 Debug 信息的输出及调用校验，就被做成了layer。
validationLayer 的启用：

  • 增加对 validationLayer 的开启

信息级别：

信息分类：

  • VK_DEBUG_UTILS_MESSAGE_TYPE_GENERAL_BIT_EXT：发生了一些与规范和性能无关的事情
  • VK_DEBUG_UTILS_MESSAGE_TYPE_VALIDATION_BIT_EXT：出现了违反规范的情况或发生了一些可能的错误
  • VK_DEBUG_UTILS_MESSAGE_TYPE_PERFORMANCE_BIT_EXT：进行了可能影响性能的行为

2. Vulkan 的基本单元封装

2.1 Physical Device

VkPhysicalDevice：物理设备
vkEnumeratePhysicalDevices：枚举出本地所有的显卡
VkPhysicalDeviceProperties：显卡的的属性结构体
vkGetPhysicalDeviceProperties：获取显卡属性并将其填充至属性结构体
VkPhysicalDeviceFeatures：显卡的特性结构体
vkGetPhysicalDeviceFeatures：获取显卡的特性，并将其填充至显卡特性结构体

2.2 队列族与逻辑设备

队列族：是一个队列的类型，类似于C++当中的class，只代表类型，不代表具体的
队列。其中规定了本类型队列应该满足的队列功能。

渲染队列：渲染相关的命令，送入该队列，可被执行

显示队列：渲染完毕的图片，形成命令，送入该队列，可以执行显示功能

传输队列：比如Buffer之间的数据拷贝，CPU GPU之间的数据拷贝，可执送入该队列

命令队列：由命令队列族为类型，创建的队列实例，驱动只允许一个队列族创建较为少量的队列，所以可以使用多个二级命令提交到一个主命令中，然后提交主命令到一个队列的方式，即多线程渲染秘籍。

2.5 交换链

逻辑设备扩展：有很多设备，比如云渲染不需要交换链向屏幕显示，所以其被设计为扩展，VK_KHR_SWAPCHAIN_EXTENSION_NAME

查询表面支持细节：

  • 基础特性：最小/最大图像数量， 最小/最大图像宽高(VkSurfaceCapabilitiesKHR)(vkGetPhysicalDeviceSurfaceCapabilitiesKHR)
  • 表面格式：像素格式（RGB RGBA）， 颜色空间（SRGB-Gamma校正后的颜色）(VkSurfaceFormatKHR)(vkGetPhysicalDeviceSurfaceFormatsKHR)
  • 可用呈现模式：先进先出，立即模式等(VkPresentModeKHR)(vkGetPhysicalDeviceSurfacePresentModesKHR)

表面格式储备知识：

  • 颜色空间：由于各个显示器对于RGB颜色值的反应不同，比如红色为1.0应该比0.5更加量一倍，但是在显示器上并不是这样的，而是呈现2.2次幂的升高曲线

  • 解决：在FragmentShader输出的时候，对每个颜色进行一次1/2.2次幂运算，然后经过屏幕的2.2次幂的指数操作，整体就会呈现线性，即GAMMA校正

  • sRGB颜色空间：指的是当前这张图片当中的颜色，在存储的时候就已经被做了取1/2.2次幂的操作

在Vulkan当中，一张图片有两个问题：

  • 对于读入一张图片，这张图片到底是什么颜色空间的图片，如何使用？
  • 对于渲染完毕的一张图片，显示器如何解释并且显示这张图片？

先看输入一张图片：
在设计软件做一张图片的时候，其实设计人员看到的是设计软件做了gamma校正后的颜色，保存一张
图片为sRGB格式后，其实也是gamma校正后的像素值，所以图片的格式为sRGB颜色空间。这样的图片，直接打开就能正确显示颜色，因为显示器会做一次2.2次幂的自然展示，颜色就可以线性。
对于Vulkan：
输入的图片如果是sRGB格式，那么就需要告诉Vulkan，使用VK_FORMAT_R8G8B8A8_SRGB的读取格式对其进行记录，之后在Shader当中对其采样使用的时候，采到的RGB数据就是从sRGB转到线性空间的值了。此时对其进行各类运算是没问题的

交换链当中的图片：
在FragmentShader最后，会向Buffer输出每个像素的颜色值，此时：
如果Buffer图片是VK_FORMAT_R8G8B8A8_UNORM，那么输出什么RGB，就是什么样
如果Buffer图片是VK_FORMAT_R8G8B8A8_SRGB，那么输出的RGB会被做1/2.2幂运算

Vulkan设置颜色空间：
对于显示表面的图片，还需要设置其颜色空间，即对于当前图片，显示引擎如何对待。一般情况下都会设置VK_COLOR_SPACE_SRGB_NONLINEAR_KHR，这样显示引擎就知道输入的是sRGB图片，直接进行显示就可以了。

TIPS：
如果显示格式设置为VK_FORMAT_R8G8B8A8_UNORM，而颜色空间为
VK_COLOR_SPACE_SRGB_NONLINEAR_KHR，显示的颜色其实是不对的，因为真正格式并不是sRGB

显示模式：

用来规定，当显示队列当中如何处理多帧输入的显示问题。

  • VK_PRESENT_MODE_IMMEDIATE_KHR：渲染完毕的图片，会立即送去显示，因此有可能产生撕裂
  • VK_PRESENT_MODE_FIFO_KHR：渲染完毕的图片，会立即送入队列，先到的会先显示，如果队列满了，渲染程序就会阻塞等待。每次在垂直消隐期间才会更新一次显示图片。
  • VK_PRESENT_MODE_FIFO_RELAXED_KHR：与上方一样，不同的是，如果队列空置，新图片会直接送去显示，不用等待垂直消隐，会造成撕裂。
  • VK_PRESENT_MODE_MAILBOX_KHR：与第二个一样，但是新来的图片会替换队列当中的图片，这样渲染效率会提高，不用阻塞等待
