2.1 给实例添加扩展

  • glfwGetRequiredInstanceExtensions(uint32_t *count)
    • 获取 glfw 所需要的扩展
    • VK_KHR_surface 和 VK_KHR_win32_surface
  • VK_EXT_debug_utils
    • 提供更丰富的调试和诊断信息的一个扩展
    • 用到的结构体：
      • DebugUtilsMessengerCreateInfoEXT
      • DebugUtilsMessengerCreateFlagsEXT
      • VkDebugUtilsMessageSeverityFlagBitsEXT 或 DebugUtilsMessageSeverityFlagBitsEXT
      • VkDebugUtilsMessageTypeFlagsEXT 或 DebugUtilsMessageTypeFlagBitsEXT
      • VkDebugUtilsMessengerCallbackDataEXT

添加了 VK_EXT_debug_utils 后我们还需要配置回调函数：

  • vk::DispatchLoaderDynamic
    • 是 Vulkan-Hpp 中的一个辅助类，用于动态加载 Vulkan 函数指针。它允许在运行时从 Vulkan 库中加载函数，而不是在编译时静态链接。

2.2 给实例添加验证层

验证层（Validation Layers）是用于帮助开发者在开发和调试阶段捕捉错误和潜在问题的工具。验证层会在 Vulkan API 调用之前和之后插入额外的代码，以检查应用程序的行为是否符合 Vulkan 规范，并报告任何违规行为。

3. 挑选一个物理设备

所谓得物理设备，就是指CPU。它是不可以创建和销毁，我们只能根据主机得配置挑选合适得设备。

  • Instance::enumeratePhysicalDevices( Dispatch const & d )
    • 枚举所有得物理设备，并返回一个 std::vector<vk::PhysicalDevice>
  • getProperties()
    • 这是 PhysicalDevice 类中的一个函数
    • 返回的是 PhysicalDeviceProperties，包含了deviceType、deviceName等字段。
    • deviceType 的类型是 PhysicalDeviceType，这是一个枚举类
  • enumerateDeviceExtensionProperties()
    • 这也是 PhysicalDevice 的函数
    • 返回的是一个 ExtensionProperties 的集合，它保存了这个设备支持的所有扩展
      • 这个结构体有两个字段：extensionName, specVersion
