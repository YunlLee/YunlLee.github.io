git 工作区域和工作状态

git 的本地数据管理分为三个区域

  • 工作区：我们电脑上的目录，在资源管理器里面能够看到的文件夹就是工作区。
  • 暂存区：临时存储即将提交到Git仓库的修改内容，是进行版本控制时非常重要的一个区域。
  • 本地仓库：就是我们用 “git init” 创建的仓库。它包含了完整的项目历史和元数据，是git存储代码和版本信息的主要位置。

在我们修改完工作区的代码时，需要使用 git add 将代码添加到暂存区，然后再将暂存区的内容提交到本地仓库中git commit。
这个过程我们可以使用 git 提供的命令来查看、比较或撤销、修改内容来保证代码的准确性和完整性。

git 中文件也有在不同状态

添加和提交文件

  • git init 创建仓库
  • git status 查看仓库状态
  • git add 添加到暂存区
  • git rm --cached 把添加到暂存区的文件拿回来
  • git commit 提交，只会提交暂存区的文件，不会提交没有跟踪的文件
  • git commit -a 同时执行暂存和提交两个动作。
  • git log [–oneline(查看简洁的提交记录)] 查看仓库的提交历史
  • git ls-files 查看仓库的状态。

回溯版本

在开发中我们经常需要回退到之前的某一个版本 git reset， reset命令有三种用法：

  • git reset --soft <要回退的版本id(git log查看)> soft 参数，在回退到上一个版本时，工作区和暂存区的内容都不会被清除。
  • git reset --hard HEAD^ 回退到上一个版本，并清多余空所有内容。
  • git reset --mixed HEAD^ 回退到上一个版本，清空多余暂存区内容

误操作了也不用担心，git 中的所有操作都是可以回溯的。

  • git reflog 查看操作的历史记录
  • git reset -hard [误操作的版本号]

使用 git diff 查看差异

  • git diff 什么都不加会比较工作区和暂存区之间的差异内容，他会显示更改的文件以及更改的详细信息。
  • git diff HEAD 比较工作区和版本库之间的差异。
  • git diff cached 比较暂存区和版本库之间的差异。
  • git diff [版本id_1] [版本id_2] 比较两个版本之间的内容。
  • git rm 删除文件。
  • git rm --cached [文件名] 只删除版本中文件

解决合并冲突

如果两个分支没有重合的内容的话合并分支就非常的简单，git 会自动完成合并。但如果两个分支修改了同一个文件的同一行代码，git 就不知道该保留那个分支修改的内容了，也就产生了冲突，这个时候就需要我们手动来解决冲突。

除了 merge 以外，还可以使用 rebase 完成合并

如果在 dev 支执行 git rebase main，那么 dev 分支就会变基到 main 分支上

如果在 main 分支上执行 git rebase dev, 那么 main 就会变积到 dev 上。
